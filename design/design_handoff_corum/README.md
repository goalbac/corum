# Handoff: Corum — 사단법인 내부 웹사이트 (일반 사용자 영역)

## Overview
Corum은 사단법인(비영리 단체·협회) 내부용 게시판 중심 협업 플랫폼입니다. 이 핸드오프는 **일반 사용자 영역**의 신규 UI/UX 디자인을 담습니다: 앱 셸(헤더+상단메뉴+사이드바+콘텐츠+푸터), 대시보드, 게시판 3종(POST/GALLERY/DOCUMENT), 게시글 상세/작성, 캘린더. 라이트/다크 모드 전환을 기본 구조로 포함합니다.

> 아직 미포함(후속 예정): 쪽지함, 마이페이지, 로그인/회원가입, **관리자 패널(별도 레이아웃)**.

## About the Design Files
이 번들의 `Corum.dc.html`은 **HTML로 만든 디자인 레퍼런스(프로토타입)**입니다 — 의도한 외관과 동작을 보여주는 목적이며, 그대로 복사해 배포하는 프로덕션 코드가 아닙니다. 이 파일은 스트리밍 렌더링용 단일 파일 컴포넌트(템플릿 + 로직 클래스 + 인라인 스타일)로 작성되어 있으니, **코드를 그대로 가져오지 말고** 대상 코드베이스(React/Vue/Svelte 등)의 기존 패턴·라이브러리로 **이 디자인을 재현**하세요. 기존 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하면 됩니다.

읽는 법: 화면 마크업은 `<x-dc>...</x-dc>` 템플릿(상단), 데이터·상호작용 로직은 `class Component extends DCLogic { ... }` 안의 `renderVals()`에 있습니다. 스타일은 전부 인라인이며 CSS 변수(`--primary` 등)를 토큰으로 사용합니다.

## Fidelity
**High-fidelity (hifi)** — 최종 색상·타이포·간격·상호작용을 포함한 픽셀 단위 목업입니다. 색/폰트/스페이싱/반경/그림자 값을 그대로 따르되, 대상 코드베이스의 기존 컴포넌트·아이콘·디자인 시스템 위에서 재현하세요.

데스크톱 우선 설계입니다(모바일 반응형은 후속). 한국어 단일 서비스.

### Colors — Dark (런타임 오버라이드)
`--bg #0e1117` · `--surface #161b22` · `--surface-2 #1d242e` · `--surface-3 #26303b` · `--border #28313d` · `--border-strong #36414f` · `--text #e7eaf0` · `--text-2 #9aa4b3` · `--text-3 #6c7686` · `--primary #5e87ee` · `--primary-strong #7ba0f2` · `--primary-weak #1a2540` · `--accent #37b585` · `--accent-weak #14302a` · `--danger #e2615b` · `--new #f06a52`

> **다크 모드 구현 주의:** 토큰을 정의하는 엘리먼트가 동시에 `color: var(--text)`를 소비하면, 같은 엘리먼트의 self-reference가 토큰 변경 시 재계산되지 않는 브라우저 이슈가 있습니다. 프로토타입은 토글 시 루트의 `color`를 리터럴(`#e7eaf0`/`#1a1e27`)로도 직접 세팅해 회피합니다. 대상 코드베이스에서는 **토큰은 wrapper/`:root`에, `color` 소비는 내부 엘리먼트에** 두는 구조를 권장합니다(또는 `data-theme` 속성 + 클래스 토글).

### Typography
- Family: **Pretendard** (Variable, CDN: `cdn.jsdelivr.net/gh/orioncactus/pretendard`). 단일 패밀리.
- Scale: Display 40/800, Heading 25–28/800, Title 19–20/700, Body 15–15.5/400–500, Caption 12–13/500–600
- `letter-spacing` 제목류 약 `-0.02~ -0.03em`, `line-height` 본문 1.6–1.9

### Radius / Shadow / Spacing
- Radius: 버튼 9px, 카드 14–16px, 칩/pill 6px·14–18px(둥근), 아이콘버튼 7–10px, 아바타 50%
- `--shadow-sm`: `0 1px 2px rgba(20,30,55,0.06)` / `--shadow`: `0 1px 2px rgba(20,30,55,0.05),0 4px 14px rgba(20,30,55,0.06)`
- Spacing: 콘텐츠 패딩 30/42px, 카드 내부 16–24px, 위젯 간격 18px, 폼 필드 간격 16–18px
- 아이콘: Lucide 스타일 라인 아이콘(stroke 1.7–1.9, viewBox 24)

### Tweakable props (디자인 컴포넌트 prop)
- `primary` (color, 기본 `#2f5fd6`, 옵션 `#2f5fd6 #1f6f5b #7a4ad6 #b3541e`) — 브랜드 컬러 토큰. `color-mix`로 strong/weak 파생.
- `defaultDark` (boolean) — 다크로 초기 진입.

## Layout — 앱 셸 (모든 일반 사용자 화면 공통)

```
┌───────────────────────────────────────────────┐
│ HEADER (64px, sticky, surface, border-bottom)  │
│ 로고 | 상단메뉴 4개 | (spacer) | 검색·알림·쪽지·테마·프로필 │
├──────────┬────────────────────────────────────┤
│ SIDEBAR  │ MAIN (flex:1, overflow-y:auto,      │
│ 268px    │       padding 30/42px)              │
│ 트리메뉴 │  └ 화면별 콘텐츠                     │
├──────────┴────────────────────────────────────┤
│ FOOTER (surface, border-top, 22/42px)          │
└───────────────────────────────────────────────┘
```

- **Header**: 좌측 로고(34px 라운드 스퀘어 primary + "코럼"/"한국공익협회"), 상단 메뉴 **최대 4개**(협회소개·공지·자료·활동·일정·소통공간) — 활성 시 하단 3px primary 인디케이터 + 텍스트 진하게, NEW 메뉴엔 6px 코럴 점. 우측: 검색·알림벨(카운트 뱃지)·쪽지(카운트)·구분선·테마토글(해/달)·프로필(아바타+이름+역할+chevron).
- **Sidebar**: 활성 상단메뉴의 하위 트리(계층 3~4뎁스). 노드 유형 = 폴더(chevron+folder 아이콘, 클릭 토글)/페이지(file 아이콘)/외부링크(external 아이콘). 들여쓰기 = `12 + depth*16`px. 활성 페이지 = `--primary-weak` 배경 + `--primary` 텍스트 + weight 700. 권한 없는 메뉴 = 자물쇠 아이콘 + `--text-3`. 하단 "내 권한" 카드(읽기/쓰기/댓글/다운로드 칩).
- **Footer**: 좌측 단체 정보(명/주소/고유번호), 우측 링크(이용약관·개인정보처리방침·문의).

상단 메뉴 클릭 → 사이드바 트리 전체 교체. 사이드바 게시판 클릭 → 해당 게시판 화면으로 라우팅(아래 매핑).

## Screens / Views

### 1) 대시보드 (Home)
- **목적**: 로그인 후 첫 화면. 위젯 집합. (관리자가 위젯 순서·표시여부 설정 — 우상단 "위젯 맞춤" 진입점)
- **레이아웃**: 인사말 헤더 → 통계 스트립(4열 grid) → 본문 2열 grid `1.7fr / 1fr`.
- **위젯**:
  - 통계 카드 ×4: 전체 회원/오늘 방문/최근 새 글/안 읽은 쪽지. 큰 숫자(26/800) + 증감(accent).
  - **주요 소식 슬라이더**(좌측): 220px 배너, 좌우 버튼+도트로 슬라이드 전환, 하단 scrim+텍스트. 이미지 영역은 줄무늬 placeholder(실데이터로 교체).
  - **최신 글**(좌측): 공지사항/보도자료/자유게시판 탭 전환, 행=공지뱃지/제목/NEW점/댓글수/날짜.
  - **이번 달 일정**(우측): 미니 월 캘린더(오늘 강조, 일정 색상 점) + 다가오는 일정 리스트.
  - **회원 현황**(우측): 총원 + 정회원/일반/대기 분포 막대.
  - **바로가기**(우측): 외부 링크 2열 그리드.

### 2) 게시판 — POST (일반 게시판)
- **레이아웃**: breadcrumb → 제목+건수+글쓰기버튼 → 툴바(카테고리 칩 + 검색 + 정렬) → 테이블 카드 → 페이지네이션.
- **테이블 컬럼**: `70px 1fr 100px 76px 64px` = 번호/제목/작성자/작성일/조회.
- 상단 **공지 고정** 행 = `--primary-weak` 배경 + "공지" 뱃지(번호 대신). 제목 셀 = 카테고리 칩 + 제목(공지는 weight 700) + 댓글수 `[n]`(primary) + 첨부 클립 + NEW("N") 뱃지.
- **행 클릭 → 게시글 상세**.

### 3) 게시판 — GALLERY (이미지 그리드)
- **레이아웃**: 헤더(사진 올리기) → 툴바 → **3열 카드 그리드**(gap 16) → 페이지네이션.
- 카드: 4:3 썸네일(줄무늬 placeholder, NEW 코너 뱃지) + 제목(1줄 클램프) + 메타(작성자·날짜) + 우측 리액션(❤️ n)·댓글(n).

### 4) 게시판 — DOCUMENT (자료실)
- **레이아웃**: 헤더(파일 업로드) → **폴더 카드 4열**(활성=primary 틴트) → 현재 폴더 헤더+검색 → 파일 테이블 카드.
- **테이블 컬럼**: `1fr 96px 80px 100px 80px 70px` = 파일명/버전/크기/등록자/등록일/받기.
- 파일명 셀: 확장자 색상 아이콘(PDF 빨강 `#d6453f`, DOCX 파랑 `#2f5fd6`, XLSX 초록 `#1f9d6b`, ZIP 회색 `#8a7f72`, HWP 청록 `#0e8a8a`, IMG 보라 `#7a4ad6`; 각 12% 알파 배경) + 파일명 + `# 카테고리`. **버전 뱃지**(history 아이콘 + `v4.2`, 클릭=버전관리). **받기**: 권한 있으면 다운로드 버튼(primary-weak), 없으면 자물쇠(비활성, `cursor:not-allowed`).

### 5) 게시글 상세
- **레이아웃**: max-width 864 중앙. 목록 back → 카테고리/NEW → 제목 → 메타 행(아바타+작성자+역할뱃지 / 날짜·조회·댓글 + 공유/수정/삭제) → 첨부 박스 → 본문 → 리액션 바 → 댓글.
- **본문**: TipTap 산출물 형태(문단/정보 박스/H 제목/리스트/인용구/이미지 placeholder).
- **이모지 리액션(핵심)**: 6종 `❤️ 👍 😄 😂 😢 😠`. 카운트>0인 것만 pill로 표시. 클릭 = 내 공감 토글(카운트 ±1, 내 공감 시 `--primary-weak` 배경 + `--primary` 보더/텍스트). "공감 추가" 버튼 → 6종 피커 팝오버.
- **댓글 3뎁스**: 들여쓰기 = `depth*36`px, 각 행 상단 보더. 아바타(운영=primary solid) + 작성자 + **역할 뱃지**(운영=primary, 정회원=accent, 일반회원=text-3) + 날짜 + 본문 + 댓글 리액션 pill(정적) + 답글 버튼. **삭제 댓글** = "삭제된 댓글입니다." 이탤릭 + 휴지통 아이콘. 상단에 작성 입력창(아바타+textarea+등록).

### 6) 게시글 작성 (TipTap 에디터)
- **레이아웃**: max-width 920. 폼 카드 안에: 옵션 행(카테고리 select + 공지고정/댓글허용 토글) → 제목 input(하단 라인) → **에디터 박스** → 첨부 드롭존+칩 → 태그 → 하단 액션(취소/임시저장/등록).
- **에디터 툴바**(그룹/구분선): 실행취소·다시실행 | 본문·H1·H2·H3 | 굵게·기울임·밑줄·취소선 | 글머리·번호 목록 | 인용 | 링크·이미지·표. 본문은 `contenteditable`. 버튼은 `onMouseDown`에서 `preventDefault` 후 서식 적용(선택 영역 유지). 대상 코드베이스에선 실제 **TipTap**(또는 ProseMirror 기반)으로 구현 권장.
- **토글 스위치**: 42×24 트랙, 20px 노브, on=`--primary`/off=`--border-strong`, 노브 `left` 2px↔19px transition.

### 7) 캘린더
- **레이아웃**: 헤더(월 네비 prev/next/오늘 + 월·주·일 세그먼트 + 일정 등록) → 본문 2열 `236px / 1fr`.
- **좌측 패널**: "내 캘린더" — 복수 캘린더(협회 `#2f5fd6`/교육·행사 `#1f9d6b`/이사회·총회 `#c47f1c`/봉사활동 `#7a4ad6`/공휴일 `#d6453f`). 색상 체크박스로 표시 토글(끄면 해당 일정 숨김).
- **월 뷰**: 7×6 그리드(셀 min-height 114). 요일 헤더 일=danger·토=primary. 오늘=primary 원형 번호. 공휴일(예: 6/6 현충일)=빨강 번호+칩. 일정=색상 좌측보더 칩(시간+제목), 4개 이상이면 "+N개 더보기".
- **주 뷰**: `56px + 7열` 헤더(요일+날짜, 오늘 원형) + 시간 그리드(08–19시, 시간당 52px, hour line 배경). 일정 = 절대위치 블록(top=(시작-8)*52, height=(길이)*52-4, 색상 좌측 보더 + 12% 알파 배경).
- **일 뷰**: 동일 시간 그리드, 단일 컬럼.

### (참고) 디자인 가이드 화면
프로토타입 내 "디자인 가이드" 화면 = 팔레트/타이포/컴포넌트(버튼·뱃지·리액션 pill·입력·탭) 시각화. 구현 대상 아님(레퍼런스).

## Interactions & Behavior
- **테마 토글**: 헤더 해/달 버튼 → 라이트/다크. 모든 토큰 동시 전환, `transition: background/color .25s`.
- **상단 메뉴 전환**: 사이드바 트리 교체.
- **사이드바 라우팅**: 게시판 유형(boardType)에 따라 화면 매핑 — `POST→게시판목록`, `GALLERY→갤러리`, `DOCUMENT→자료실`, `CAL→캘린더`. 폴더 노드=확장 토글, 외부링크=새 탭(현재 no-op).
- **게시판 행 클릭 → 상세**, 목록/상세의 **글쓰기·수정 → 작성 화면**.
- **리액션**: 토글(±1, 내 공감 강조) + 피커 팝오버.
- **에디터**: contenteditable + 서식 명령. 토글 스위치/탭/카테고리 칩 활성 상태.
- **캘린더**: 뷰 전환(월/주/일), 캘린더 표시 토글로 이벤트 즉시 필터.
- **Hover**: 버튼/행/카드에 `--surface-2` 배경 또는 보더 강조. 입력 focus=`--primary` 보더.
- 페이지네이션: 활성 페이지 primary solid.

## State Management
프로토타입의 단일 컴포넌트 상태(대상 코드베이스에선 라우트/스토어/컴포넌트 상태로 분배):
- `screen` — 현재 화면(home/post/gallery/document/detail/write/calendar). 실제로는 **URL 라우팅**으로.
- `dark` — 테마. localStorage 또는 시스템 설정 연동 권장.
- `activeMenu` / `expanded{}` / `current` — 상단메뉴 인덱스 / 사이드바 폴더 펼침 / 활성 페이지 id.
- `slide`, `feedTab` — 대시보드 슬라이더/탭.
- `reactions{emoji:{count,mine}}`, `showPicker` — 게시글 리액션.
- `optNotice`, `optComment` — 작성 옵션 토글.
- `calView`('month'|'week'|'day'), `calOn{calendarId:bool}` — 캘린더 뷰/표시.

데이터 페칭(실서비스): 게시글 목록/상세(페이징·검색·카테고리), 첨부 다운로드(권한 체크), 댓글 트리(3뎁스, 삭제 상태), 리액션 집계/토글, 캘린더 이벤트(복수 캘린더·반복·한국 공휴일), 회원/통계, 위젯 구성(관리자 설정). 알림은 SSE + Web Push + 이메일 3채널.
